## RASPBERRY PI PLAYER SYSTEM

The player must be optimized for Raspberry Pi devices.

Expected environment:

Raspberry Pi OS
Chromium kiosk mode
Autostart on boot

Player URL example:

http://server/player

PLAYER FEATURES

Automatic device registration
Device heartbeat monitoring
Local asset caching
Offline playback if network lost
WebSocket live updates
Low CPU usage
Hardware video acceleration support

PLAYER BOOT FLOW

1. Raspberry Pi boots
2. Chromium launches in kiosk mode
3. Player loads
4. Player registers device

POST /api/screens/register

Server returns:
deviceToken
assignedPlaylist
layout

5. Player opens WebSocket
6. Player downloads assets and caches them locally
7. Player renders layout zones

If connection lost:
player continues using cached content.
